## Overview

The `signTransfer` function creates an EIP-712 signature for transferring tokens between sub-accounts on the GRVT Exchange. This signature is required when submitting a transfer request to the API.

## Function Signature

```typescript
signTransfer(
  transfer: IApiTransferRequest,
  wallet: Wallet,
  env: EGrvtEnvironment,
  options?: ISigningOptions
): Promise<ISignature>
```

## EIP-712 Structure

The function signs a structured message according to EIP-712 with the following schema:

**Domain:**
- `name`: "GRVT Exchange"
- `version`: "0"
- `chainId`: Environment-specific chain ID

**Primary Type:** `Transfer`

**Message Fields:**
- `fromAccount` (address): Source account ID
- `fromSubAccount` (uint64): Source sub-account ID
- `toAccount` (address): Destination account ID
- `toSubAccount` (uint64): Destination sub-account ID
- `tokenCurrency` (uint8): Currency enumeration index
- `numTokens` (uint64): Token amount in micro-units (6 decimal places)
- `nonce` (uint32): Unique nonce
- `expiration` (int64): Expiration timestamp in nanoseconds

## Example

```typescript
import { signTransfer } from '@grvt/sdk';
import { Wallet } from 'ethers';
import { EGrvtEnvironment } from '@grvt/sdk/config';

// Create wallet from private key
const wallet = new Wallet('0x...');

// Prepare transfer request
const transferRequest = {
  from_account_id: '0x1234...',
  from_sub_account_id: '1',
  to_account_id: '0x5678...',
  to_sub_account_id: '2',
  currency: 'USD',
  num_tokens: '100.50' // Transfer 100.50 tokens
};

// Sign the transfer
const signature = await signTransfer(
  transferRequest,
  wallet,
  EGrvtEnvironment.PROD
);

// Use signature in API request
const request = {
  ...transferRequest,
  signature
};

// Submit to API
await client.transfer(request);
```

## Example with Custom Options

```typescript
import { signTransfer, GenerateNonce, GenerateExpiration } from '@grvt/sdk';

// Custom nonce and expiration
const signature = await signTransfer(
  transferRequest,
  wallet,
  EGrvtEnvironment.PROD,
  {
    nonce: GenerateNonce(),
    expiration: GenerateExpiration(48) // 48 hours
  }
);
```

## Validation

The function validates:
- `num_tokens` is provided (throws error if missing)
- `num_tokens` has at most one decimal point
- `num_tokens` has at most 6 decimal places
- Custom `options` are valid (if provided)

## Errors

- Throws `Error` if `num_tokens` is not provided
- Throws `Error` if `num_tokens` has more than one decimal point
- Throws `Error` if `num_tokens` has more than 6 decimal places
- Throws `Error` if `options` validation fails
